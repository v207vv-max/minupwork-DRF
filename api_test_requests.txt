How to test API
================

1. Сначала создать пользователей:
   - client
   - freelancer
2. Подтвердить регистрацию через verification code.
3. Затем получить JWT token.
4. Для защищённых endpoints добавить заголовок:

Authorization: Bearer ACCESS_TOKEN

5. Базовый адрес для локального тестирования:

http://127.0.0.1:8000

6. Рекомендуемая последовательность тестирования:
   - Register client
   - Verify signup code
   - Login client
   - Register freelancer
   - Verify signup code
   - Login freelancer
   - Client creates project
   - Freelancer creates bid
   - Client views project bids
   - Client accepts bid
   - Client/Freelancer view contracts
   - Client finishes contract
   - Client creates review
   - Client/Freelancer test chat

7. Важно:
   - endpoints `/api/accounts/*`, `/api/projects/*`, `/api/bids/*`, `/api/contracts/*`, `/api/reviews/*`, `/api/chat/*`
     в основном требуют JWT, кроме signup/login/password reset/token endpoints.
   - Swagger UI: `/api/docs/`
   - OpenAPI schema: `/api/schema/`


Authentication
================

Register
--------------------
Method: POST
URL: /api/accounts/signup/
JWT required: No

Body:
{
    "username": "client1",
    "role": "client",
    "email": "client1@example.com",
    "phone_number": "",
    "preferred_contact_method": "email",
    "password": "client12345",
    "confirm_password": "client12345"
}

Response example:
{
    "message": "Verification code was sent successfully.",
    "pending_user_id": 1,
    "user": {
        "id": 1,
        "username": "client1",
        "email": "client1@example.com",
        "phone_number": null,
        "role": "client",
        "bio": "",
        "image": null,
        "preferred_contact_method": "email",
        "is_email_verified": false,
        "is_phone_verified": false,
        "created_at": "2026-03-15T12:00:00Z",
        "updated_at": "2026-03-15T12:00:00Z"
    }
}

Comment:
- Сохрани `pending_user_id` для следующего шага.
